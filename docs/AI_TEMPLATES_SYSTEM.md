# AI-Generated Templates System 🤖

## Overview

A **self-improving template library** powered by Claude AI + human curation. Instead of static templates, Work Shelf now generates **custom project templates** based on user interests, with staff review to ensure quality.

## The Workflow

### 1️⃣ **User Creates Group**
- User enters interests/tags: `["science fiction", "worldbuilding", "collaborative storytelling"]`
- Can be freeform or selected from suggestions

### 2️⃣ **AI Generates Templates** (Real-time)
- Claude analyzes interests
- Generates 10 custom templates on-the-fly
- Examples for sci-fi + worldbuilding:
  * "Sci-Fi Universe Bible"
  * "Collaborative Story Arc Planner"
  * "Character Database"
  * "Technology & Magic Systems Guide"
  * "Timeline & Historical Events"

### 3️⃣ **Staff Review Dashboard** (Kit)
- See all AI-generated templates (pending review)
- Three actions:
  * **✅ Adopt** - Approve as-is
  * **✏️ Edit** - Refine prompts/sections (then approve)
  * **❌ Reject** - Not suitable
- Preview sections and AI prompts before deciding

### 4️⃣ **Template Evolution**
- Once approved → becomes canonical `ProjectTemplate`
- Linked to original interests via `template_interest_mappings`
- Next user with similar interests → gets the **improved, approved version** (no regeneration needed)
- Templates get better over time as Kit curates them

## Database Schema

### `ai_generated_templates` Table
```sql
CREATE TABLE ai_generated_templates (
    id SERIAL PRIMARY KEY,
    name VARCHAR,
    slug VARCHAR UNIQUE,
    description TEXT,
    category VARCHAR,  -- auto-categorized by AI
    icon VARCHAR,      -- emoji suggested by AI
    
    -- What interests triggered this?
    source_interests VARCHAR[],  -- e.g., ['science fiction', 'worldbuilding']
    
    -- AI metadata
    ai_model VARCHAR,              -- 'claude-3.5-sonnet'
    ai_prompt_used TEXT,           -- the prompt sent to Claude
    generation_timestamp TIMESTAMP,
    
    -- Template structure (generated by AI)
    sections JSONB,  -- full hierarchical structure with prompts
    
    -- Review status
    status VARCHAR,  -- 'pending', 'approved', 'rejected', 'edited'
    reviewed_by INTEGER REFERENCES users(id),
    reviewed_at TIMESTAMP,
    review_notes TEXT,
    
    -- If Kit edited it
    edited_sections JSONB,  -- Kit's improved version
    
    -- Usage tracking
    times_suggested INTEGER DEFAULT 0,
    times_selected INTEGER DEFAULT 0,
    groups_using INTEGER DEFAULT 0,
    
    -- Links
    requested_by_group INTEGER REFERENCES groups(id),
    approved_template_id INTEGER REFERENCES project_templates(id)
);
```

### `template_interest_mappings` Table
```sql
CREATE TABLE template_interest_mappings (
    id SERIAL PRIMARY KEY,
    template_id INTEGER REFERENCES project_templates(id),
    interest VARCHAR,
    relevance_score FLOAT DEFAULT 1.0,
    
    UNIQUE (template_id, interest)
);
```

### `ai_generation_logs` Table
```sql
CREATE TABLE ai_generation_logs (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    group_id INTEGER REFERENCES groups(id),
    interests VARCHAR[],
    ai_model VARCHAR,
    prompt_sent TEXT,
    response_received TEXT,
    templates_generated INTEGER,
    generation_time_ms INTEGER,
    error_message TEXT,
    success BOOLEAN,
    created_at TIMESTAMP
);
```

### `groups` Table Update
```sql
ALTER TABLE groups ADD COLUMN interests VARCHAR[];
```

## API Service

### `AITemplateGenerator` Class

```python
class AITemplateGenerator:
    async def generate_templates_for_interests(
        db: AsyncSession,
        interests: List[str],
        user_id: Optional[int] = None,
        group_id: Optional[int] = None,
        num_templates: int = 10
    ) -> List[AIGeneratedTemplate]
```

**What it does:**
1. Builds smart prompt for Claude based on interests
2. Calls Claude API (`claude-3-5-sonnet-20241022`)
3. Parses JSON response
4. Creates `AIGeneratedTemplate` records (status='pending')
5. Logs generation to `ai_generation_logs`

```python
async def check_existing_templates(
    db: AsyncSession,
    interests: List[str],
    status: str = "approved"
) -> List[ProjectTemplate]
```

**What it does:**
1. Checks if approved templates already exist for these interests
2. Uses `template_interest_mappings` for smart matching
3. Returns existing templates (avoids regeneration)

```python
async def approve_template(
    db: AsyncSession,
    ai_template_id: int,
    reviewer_id: int,
    edits: Optional[Dict[str, Any]] = None,
    review_notes: Optional[str] = None
) -> ProjectTemplate
```

**What it does:**
1. Converts AI template → canonical `ProjectTemplate`
2. Creates `TemplateSection` records (with nesting support)
3. Creates `TemplateInterestMapping` entries
4. Updates AI template status to 'approved'
5. Links them via `approved_template_id`

## Why This is Brilliant 🌟

### 1. **Personalized from Day One**
Static templates are generic. AI templates are **tailored to user interests**.

### 2. **Human-in-the-Loop Quality**
AI generates, Kit curates. Best of both worlds:
- Speed of AI
- Quality control of human review

### 3. **Self-Improving Library**
Every approval makes the system smarter:
- First user with "sci-fi + worldbuilding" → Claude generates
- Kit reviews and improves → becomes canonical
- Next user with same interests → gets Kit's improved version
- No wasted AI tokens, better user experience

### 4. **Data-Driven Template Creation**
Kit sees what people actually want:
- "100 users searched for 'D&D campaign' but no approved template yet"
- Prioritize building templates users are actively requesting
- Build what matters, not what we guess

### 5. **Scalability**
- First 1,000 users → lots of AI generation
- Next 10,000 users → mostly approved templates (Kit already curated)
- Eventually: comprehensive library with minimal ongoing AI cost
